# glm_usage 项目规范

## 代码风格

- 单文件代码行数尽量不超过 180 行，最多不超过 200 行
- 需要拆分和做抽象，不要堆在一个文件里
- 模块职责清晰：config / client / display / commands 各管各的

## 时区规范 ⚠️

- **Z.ai API 所有时间均为 UTC+8**（入参和返回值都是）
- 代码中统一使用 `UTC8 = timezone(timedelta(hours=8))` 常量
- **禁止使用 `timezone.utc` 或 `datetime.utcnow()`** 与 Z.ai API 交互
- `datetime.now()` 始终用 `_now()`（返回 UTC+8）
- `fromtimestamp()` 始终传 `tz=UTC8`
- 显示给用户的时间也用 UTC+8

## Git 管理

- 使用 git 管理代码，及时提交
- 根据需要进行代码变更的拆分（一个逻辑变更一次提交）

## 功能要求

- 所有子命令必须支持 `--json` 参数，以 JSON 形式输出原始数据

## GLM Coding Plan 限额规则

### 计量单位

- 单位是 **Prompts（请求次数）**，不是 Tokens
- 拥有固定的基础请求次数，但单次对话扣除的 Prompt 数量会根据模型版本和是否高峰期**动态变化**

### 各套餐基础请求限额

| 套餐 | 5 小时滚动限额 | 每周限额 | 每月总限额 |
|------|---------------|---------|-----------|
| Lite | ~80 次 | ~400 次 | ~1,600 次 |
| Pro | ~400 次 | ~2,000 次 | ~8,000 次 |
| Max | ~1,600 次 | ~8,000 次 | ~32,000 次 |

- 5 小时限额：滑动窗口机制，每次 Prompt 在精确的 5 小时后单独恢复
- 每周限额：固定周期点（通常周一）统一重置

### 动态消耗倍率

- **高峰期**: UTC+8 14:00–18:00（即 UTC 06:00–10:00）
- **高级模型**: GLM-5.1、GLM-5、GLM-5-Turbo（与 Claude Opus 同级）

| 模型 | 高峰期 (UTC+8 14:00–18:00) | 非高峰期 |
|------|---------------------------|---------|
| 常规模型（GLM-4.7 等） | 1× | 1× |
| 高级模型（GLM-5.1 / GLM-5 / GLM-5-Turbo） | 3× | 2× |

- 至 2026 年 6 月底有临时福利：非高峰期调用高级模型暂按 **1×** 扣除
